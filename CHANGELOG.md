# Changelog

## v0.6.8 (2026-07-22)

### 🗂️ 保持原文件夹层级

- **新增「保持原文件夹层级」复选框**：位于命名页面排序设置下方，开启后 `classify_files` 保留源文件夹的子目录结构，`rename_files_in_folders` / `group_compress` / `write_rename_list` 均按层级目录独立处理
- **引擎 `_collect_dirs(keep_hierarchy=True)`**：返回所有包含文件的目录（含子孙目录），扁平模式仅返回直接子目录
- **引擎 `_match_rule_hierarchy`**：从当前目录逐级向上查找祖先目录匹配命名规则，确保子文件夹也能继承父级的命名规则
- **重命名→压缩序号一致**：`rename_files_in_folders` 返回 `folder_order` 字典，`group_compress` 接受可选 `folder_order` 参数并跳过重新排序，保证重命名与压缩的文件顺序完全一致
- **`_dedup_name`**：分类时检测目标文件名是否已存在，自动追加 `_1`、`_2` 后缀，避免同名覆盖
- **预检强制层级提示**：开启递归且未勾选层级时，扫描源文件夹检测同分类下的重名文件；发现重名时弹出统一风格对话框，建议启用层级模式

### 🔍 自然排序

- **新增 `_natural_key` 函数**：使用 `re.split(r'(\d+)')` 分割文件名中的数字与非数字部分，数字部分按整型比较（`1,2,3,…,10,11` 而非 `1,10,11,2,20,3,…`）
- **`SORT_FUNCS['name']` 与 `'name_desc'` 改用自然序**：所有涉及文件名排序的环节（预览、重命名、压缩）均受益

### 🐛 Bug 修复

- **`group_compress` 删除文件异常处理**：`f.unlink()` 失败时（如只读文件 PermissionError）不再中断整个压缩流程，仅输出警告并继续处理下一组
- **`-v:` 分卷参数受 `enable_volume` 控制**：未勾选「启用分卷」时不再传递 `-v:` 参数，即使分卷大小输入框有内容
- **修复 `_match_rule` 子文件夹不匹配命名规则**：层级模式下子文件夹名（如 `sub`）无法匹配以祖先文件夹名为 match_folder 的规则，改用 `_match_rule_hierarchy` 逐级查找

### 🔧 代码清理

- **版本号统一**：`app.py` 关于面板版本号改为引用 `__init__.py` 的 `__version__`（`v0.6.6` → `v0.6.8`）
- **删除未使用 import**：`app.py` 的 `_collect_dirs`、`widgets.py` 的 `QThread`、`SortZip.py` 的 `main_from_config`
- **删除死参数**：`main_from_config` 的 `on_stats` 参数（从未使用）
- **消除重复代码**：`app.py` 的 `_match_naming_rule` 与 `engine._match_rule` 逻辑完全一致，删除并统一导入
- **`sort_map` 常量化**：两处重复定义的 `sort_map` 字典提取为模块级 `SORT_MAP` 常量
- **`SEP` 移至模块顶部**：从函数群中间提到 `import` 之后
- **清理硬编码测试数据**：`cli()` 和 `SortZip.py` 中的开发路径和测试密码

## v0.1.0 (2026-06-21)

### ✨ 初始版本

首个完整可用版本，实现文件分类→重命名→分组压缩→二次打包的完整流程。

#### 功能特性

- **文件分类**：按扩展名自动归类，支持自定义文件夹名映射
- **顺序重命名**：支持按文件名或修改时间排序，自动处理命名冲突
- **分组压缩**：
  - 调用 Bandizip 命令行进行分卷压缩
  - 自动检测分卷大小（<4GB 对半切，≥4GB 每卷 4GB）
  - 支持手动指定分卷大小
  - 支持密码保护
- **二次打包**：将分卷文件再次压缩为 `.zipp` 单文件归档
- **自动清理**：可选保留或删除原始文件
- **自动窗口**：可选自动关闭 Bandizip 弹窗

#### 配置方式

- `CONFIG` 字典集中管理全部参数
- 支持 `.txt → 文档` 等自定义扩展名→目录名映射
